
#개발 환경 셋팅

  1.JDK(JAVA Development Kit) : 자바를 개발하고, 컴파일하고, 빌드하는 모든 프로그램
	LTS : 기술지원이 3년 이상이 되는 버전
	SUN(회사에서 개발)=========>ORACLE(이 인수함) (www.oracle.com)
	1.8버전까지가 무료이고   11, 17	<=== ORACLE(유료), 11, 17
	

	OPEN JDK(오라클에서 만듬)	<===무료로 사용할 수 있도록 만들어진거

	JDK는 JRE를 포함하고 있고 추가적으로 개발에 필요한 툴이 더 내장되어 있다.
	    - JRE는 JVM(자바 가상 머신) 컴파일된 바이트 코드만 실행해준다

	JDK(클라이언트에서 개발할때) <=====================> JRE(서버에서)
	
        PATH 변수 설정	:명령 프롬프트에서 모든 겨로에서 해당 프로그램을 실행
	- PATH 변수에 등록된 폴더는 모든 경로에서 해당 폴더의 프로그램을 실행할 수가 있다.
	C:\Program Files\Java\jdk1.8.0_202\bin
		java.exe,	javac.exe

        JAVA_HOME 변수 설정 : JDK 설치된 폴더를 지정 , 
	C:\Program Files\Java\jdk1.8.0_202



  2.이클립스(Eclipse) <== IDE(통합 개발 환경) : 개발을 쉽게 해주는 프로그램, 개발툴, 디버깅, 컴파일,
	-백엔드(Back-End) 개발할때 사용함, 소기업, 중소기업
	-IBM에서 만들었고 무료로 사용할 수 있는 IDE
    인텔리제이 : 유료버전 <==Back-end, Front-end, 단 커뮤니티 버전은 무료로 사용할 수 있음, 울티메이트 버전은 유로(1년 50만)
	- 대기업에서 주로 사용함(카카오, 네이버)

  3.VS Code:	<==Front-End 개발


플랫폼 :
    -Hardware 플랫폼 : 여러종류의 CPU가 있다. 
    -OS 플랫폼 : Unix, windows, Linux


C나 C++ : 완전한 기계어로 컴파일이 된다.
   -- 플랫폼마다 해당 플랫폼에 맞게 컴파일 해야 한다.
   -- 다른 플래폼에 컴파일된 파일은 작동 되지 않는다.


JAVA 는 완전한 기계어로 컴파일이 되는 것이 아니라 중간언어로 컴파일이 된다.	<===byte 코드로 컴파일이 된다. 
	Hello.java(프로그램의 소스파일)  <==== 컴파일 =======> Hello.class  (중간언어로 컴파일)	

	JVM이 설치된 windows, mac, linux 플랫폼에 종속적이지 않고 모든 곳에서 작동됨



패키지(Package) : JAVA의 소스파일의 폴더 구조를 생성함.
클래스(class)  : 개발시 자바의 소스파일을 생성
    - 클래스이름의 첫자는 반드시 영문 대문자.
